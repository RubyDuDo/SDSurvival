# V2：基础机制强化

## 设计目标

在不增加新系统的前提下，通过**信息透明化、递减收益、行动细化、动态市场、数值重平衡**五项改动，让现有系统内的决策更丰富、反馈更清晰。

V2完成后，玩家应该能感受到："这周我确实需要想一下怎么分配精力"，而不是无脑按固定顺序操作。

---

## 改动1：信息透明化

### 设计意图

玩家看不到行动的具体效果，就无法做出有依据的选择。把关键概率和边际收益直接显示出来。

### 具体规则

**行动按钮增强显示：**

学习类按钮在原有信息基础上，增加对当前求职的影响提示：

```
学习系统开发（能量 -1，进度 3/6）
  → 当前最佳岗位简历通过率：52% → 56%
```

只在玩家有活跃求职目标（已投递或有心仪岗位）时显示。没有时不显示第二行。

**投递简历子菜单增强：**

每个岗位按钮增加预估通过率：

```
[字节跳动] 初级后端 | 要求系统5 ✓ 英2 周薪$1,100  简历≈62% 面试≈38%
```

- `简历≈62%`：基于当前技能、英语、C++、作品集计算的简历通过率
- `面试≈38%`：vibe率 × 竞争率的综合通过率
- 颜色编码：≥60% 绿色，30-59% 黄色，<30% 红色

**结算面板增强：**

简历和面试结果增加原因说明：

```
❌ [初级后端] 字节跳动：简历未通过（技能差距 -2 级，英语不足）
❌ [全栈开发] 美团：面试未通过（文化匹配关未过，面试技巧偏低）
✅ [外包测试] 外包A：简历通过！已安排面试（作品集加分 +16%）
```

### UI布局

行动按钮从单行变为可选的两行模式。第二行用较小字号（14px）和灰色显示边际效果：

```
┌─────────────────────────────────────────────────────┐
│ 学习系统开发（能量 -1，进度 3/6）                       │
│   ↳ 初级后端简历通过率：52% → 56%                      │
└─────────────────────────────────────────────────────┘
```

投递子菜单的概率直接附在岗位按钮右侧，不需要额外行：

```
┌─────────────────────────────────────────────────────┐
│ [字节跳动] 初级后端 | 系统5✓ 英2    $1,100/周          │
│                           简历≈62%  面试≈38%  综合≈24% │
└─────────────────────────────────────────────────────┘
```

## 改动2：学习疲劳系统

### 设计意图

打破"每周把所有EP都砸同一个技能"的固定模式，迫使玩家在不同行动间分散精力。

### 具体规则

**每周同一技能的学习次数有递减收益：**

| 本周第N次学习同一技能 | XP获得 |
|----------------------|--------|
| 第1次 | 1.0 XP |
| 第2次 | 0.5 XP |
| 第3次及以后 | 0 XP（按钮禁用，提示"本周已疲劳"） |

- 每周重置疲劳计数
- "灵感爆发"事件效果改为：本周学习疲劳阈值+1（即第1、2、3次都是1XP，第4次才递减）
- 不同技能独立计算疲劳（学2次系统+2次面试完全正常）

### 对策略的影响

7EP的分配从"7次学同一个"变成必须分散：
- 学技能A×2 + 学技能B×2 + 做别的事×3
- 学技能A×2 + 人脉×1 + 零工×3 + 学技能B×1
- 玩家被迫在更多行动类型间做选择

### UI变化

按钮显示疲劳状态：

```
学习系统开发（能量 -1，进度 3/6）          ← 正常
学习系统开发（能量 -1，进度 3/6）⚡效率减半  ← 本周已学1次
学习系统开发 — 本周已疲劳                  ← 本周已学2次，禁用
```

## 改动3：行动细化

### 设计意图

把"学习"从单一动作拆分为多种子行动，让同一个目标有不同的实现路径。

### 具体规则

**每个技能方向（系统/应用）的学习拆分为三种方式：**

| 行动 | 消耗 | 效果 | 特点 |
|------|------|------|------|
| 看文档/教程 | 1EP | +1 XP（稳定） | 安全选择，受疲劳限制 |
| 刷题/实战练习 | 1EP | 随机获得0-2 XP（期望1.0） | 60%概率1XP，20%概率2XP，20%概率0XP。不受疲劳限制但有风险 |
| 做开源贡献 | 2EP | +1 XP + 0.5作品集进度 | 一石二鸟，但单项效率低 |

**面试技巧的学习拆分为：**

| 行动 | 消耗 | 效果 |
|------|------|------|
| 看面经 | 1EP | +1 XP（稳定），受疲劳限制 |
| 模拟面试 | 2EP | +1 XP + 下一次面试vibe率额外+10%（一次性buff） |

**英语学习拆分为：**

| 行动 | 消耗 | 效果 |
|------|------|------|
| 背单词/语法 | 1EP | +1 XP（稳定），受疲劳限制 |
| 参加英语角 | 2EP | +1英语XP + 1人脉点 |

**零工也拆分：**

| 行动 | 消耗 | 效果 | 条件 |
|------|------|------|------|
| 兼职零工 | 3EP | +$280 | 无条件 |
| 全职零工 | 5EP | +$520 | 无条件 |
| 技术私活 | 2EP | +$150 + 最高专业技能×$40 | 专业技能≥3 |

技术私活的关键设计：让高技能玩家有更高效的赚钱方式。技能5的玩家：2EP赚$350（比兼职零工3EP=$280更高效）。这让"先升技能"有了赚钱方面的回报，同时也让"升技能是为了找工作还是为了赚钱"成为一个有趣的选择。

### UI布局

行动按钮区按类别分组，用分隔线区分：

```
── 学习 ──
看文档·系统开发（能量-1，进度 3/6）
刷题·系统开发（能量-1，期望≈1XP，可能0-2）
做开源·系统开发（能量-2，+1XP +0.5作品集）
看文档·应用开发（能量-1，进度 1/2）
...
看面经（能量-1，面试技巧 进度 2/3）
模拟面试（能量-2，+1XP，下次面试vibe+10%）
背单词（能量-1，英语 进度 0/1）
参加英语角（能量-2，+1英语XP +1人脉）

── 工作 ──
兼职零工（能量-3，+$280）
全职零工（能量-5，+$520）
技术私活（能量-2，+$350）        ← 有技能时显示实际金额

── 社交与求职 ──
维护人脉（能量-1，人脉 3→4/10）
做个人项目·系统（能量-3，作品集 1→2/3）
投递简历（能量-1）
参加面试：[岗位名]（能量-2）
```

按钮过多时用折叠分组。默认展开"学习"和"工作"，"社交与求职"也展开。玩家可点击分组标题折叠/展开。

## 改动4：动态市场

### 设计意图

让外部环境变化成为影响策略的因素，增加适应性决策和重玩差异。

### 具体规则

**市场热度系统：**

每次市场刷新（每7周一次）时，除了现有的market_bias（系统/应用/均衡），额外生成一个**热门技能**：

```
热门技能池：[系统开发, 应用开发, C++, 英语, 面试技巧]
每次刷新随机选1个作为热门技能，持续到下次刷新
```

**热门技能效果：**
- 该技能相关岗位的**面试竞争率降低10%**（岗位多了，竞争缓解）
- 该技能的学习XP +0.5（市场上相关资源更丰富）
- 如果热门技能是"面试技巧"：所有面试vibe基础率+5%
- 如果热门技能是"英语"：所有岗位英语要求-1（最低0）

**市场波动事件（新增随机事件）：**

| 事件 | 效果 | 持续 |
|------|------|------|
| AI替代恐慌 | 应用方向岗位薪资-15%，系统方向不受影响 | 2周 |
| 新技术风口 | 随机一个方向岗位+1个额外高薪临时岗 | 本次刷新期间 |
| 大厂扩招 | 所有岗位竞争率-10% | 2周 |
| 裁员潮 | 所有岗位竞争率+15%，但市场上多1个低级岗位 | 2周 |

### UI变化

Header区域增加市场热度显示：

```
第 8 周 / 52 周    现金: $3,200    🔥热门：C++（竞争↓ 学习+0.5XP）
```

市场刷新时结算面板显示：

```
★ 岗位市场已刷新！
  市场偏向：系统/后端需求旺盛
  本期热门技能：C++（相关岗位竞争降低，学习更高效）
```

## 改动5：数值重平衡

### 设计意图

让Portfolio和人脉成为真正有竞争力的策略路线，而非明显的劣势选项。

### 具体调整

**Portfolio改动：**

| 项目 | 原值 | 新值 | 理由 |
|------|------|------|------|
| 消耗 | 3EP | 维持3EP（但开源贡献2EP也能涨） | 开源贡献提供了更灵活的路径 |
| 简历加成/个 | +8% | +10% | 提升收益 |
| 面试vibe倍率 | ×1.00/1.08/1.15/1.20 | ×1.00/1.10/1.18/1.25 | 3个作品集的面试加成从+20%升到+25% |
| 新增效果 | 无 | 每个作品集使技术私活收入+$50 | 作品集=能力证明=更好的私活 |

**人脉改动：**

| 项目 | 原值 | 新值 | 理由 |
|------|------|------|------|
| 内推事件权重 | 1-3点×1.5，4-6×2.5，7-9×4 | 维持不变 | 原值合理 |
| 新增被动效果 | 无 | 人脉≥3：每周有15%概率获得小道消息（提前预览下周某个随机事件） | 信息优势，帮助规划 |
| 新增被动效果 | 无 | 人脉≥5：面试时vibe基础率+5%（朋友帮你打听了面试官风格） | 直接战斗力提升 |
| 新增被动效果 | 无 | 人脉≥8：每周投简历时可额外免费投1份（不算入海投惩罚） | 效率提升 |

**零工改动：**

| 项目 | 原值 | 新值 | 理由 |
|------|------|------|------|
| 兼职零工 | 3EP→$280 | 维持不变 | 作为保底选项合理 |
| 全职零工 | 5EP→$520 | 维持不变 | 作为保底选项合理 |
| 新增：技术私活 | 无 | 2EP→$150+技能×$40 | 高技能玩家的赚钱新路径 |

### 对策略的影响

改动后三条有竞争力的路线：

**技能流**：堆专业技能 → 技术私活赚钱缓解经济压力 → 高技能硬投高级岗
**作品集流**：适度技能 + 堆作品集 → 简历通过率高 + 面试vibe高 + 私活收入更高
**社交流**：优先人脉 → 信息优势 + 面试加成 + 投递效率 + 内推机会

## 实现优先级

1. **信息透明化**（改UI显示，不碰核心逻辑）
2. **学习疲劳**（改学习结算逻辑 + 按钮状态）
3. **数值重平衡**（调数值 + 加人脉被动效果）
4. **行动细化**（拆分按钮 + 新结算逻辑）
5. **动态市场**（新系统，最后做）
