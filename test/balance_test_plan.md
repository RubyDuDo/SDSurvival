# 数值平衡测试方案

## 一、测试方法

用 Python 蒙特卡洛模拟器复刻核心游戏逻辑，每策略×方向跑 2000 局，统计结果分布。

## 二、测试策略组

| 策略 | 描述 |
|------|------|
| 纯打工 | 只做零工/外包，不学习不投简历 |
| 纯学习 | 前期全学习，后期才投简历 |
| 激进求职 | 前两周学一下就开始疯狂投简历 |
| 平衡型 | 每周学习+投简历+适当打工 |
| 外包流 | 主攻外包收入，顺带求职 |
| 大厂冲刺 | 全力练算法，只投大厂 |

每个策略 × 5个技能方向 = 30 个组合，每组合 2000 局。

## 三、核心统计指标

| 指标 | 说明 |
|------|------|
| 最终评级分布 | S/A/B/C/F 各占比 |
| 预估年收入 | 均值、中位数、标准差 |
| 首次Offer周数 | 均值、最早、最晚 |
| 总Offer数 | 平均拿到几个 |
| 破产率 | 现金<0 的比例 |
| 最终就业率 | 结束时有工作的比例 |
| 最终技能等级 | 主技能平均等级 |
| 岗位可达性 | 高级岗位实际投上的比例 |

## 四、判断"有问题"的标准

| 问题类型 | 判断条件 |
|----------|----------|
| 某策略绝对碾压 | 一个策略的 A+S 率 > 其他所有策略 20%+ |
| 某策略完全不可行 | 破产率 > 60% 或 S 率 = 0% |
| 技能方向无差异 | 5个方向同策略下结果方差 < 5% |
| 技能方向严重失衡 | 某方向年收入均值比最优方向低 30%+ |
| 高级岗不可达 | 合理策略下高级岗投递率 < 5% |
| 经济过松/过紧 | 平衡型破产率 > 40%（太紧）或 F率 = 0%（太松） |
| Offer过多/过少 | 平均 offer > 5（太容易）或 < 0.3（太难） |
| 冷启动问题 | 前3周能投的岗位数 = 0 |

## 五、额外分析

1. 经济曲线：12周现金变化，识别"必死区间"
2. 技能天花板：主技能最高等级 vs 高级岗 Lv.6 门槛
3. 工具 ROI：各工具投入产出比
4. 特质触发率：各策略下特质触发概率

## 六、产出

1. 每个策略×方向的统计表
2. 明显失衡点清单 + 调整建议
3. 关键数值敏感度分析
