# V4: 分数制竞争力系统 —— 详细设计

> 状态：设计细化
> 日期：2026-03-24

## 设计目标

用**透明可算的分数**替代当前的**概率黑盒**，让玩家能：
1. 一眼看出自己的实力
2. 一眼看出离目标的差距
3. 明确知道每个行动值多少分

## 核心公式

```
简历分 × 面试分 = 竞争力

竞争力 vs 岗位要求 + 随机对手 → 结果
```

## 四、岗位要求

每个岗位有三道门槛：简历门槛、面试门槛、竞争力门槛。

### 岗位数值表

| 岗位 | 方向 | 简历门槛 | 面试门槛 | 竞争力门槛 | 周薪 |
|------|------|---------|---------|-----------|------|
| 外包测试 | 系统 | 5 | 3 | 20 | $780 |
| 兼职前端 | 应用 | 4 | 3 | 16 | $680 |
| 初级后端 | 系统 | 14 | 10 | 180 | $1100 |
| 全栈开发 | 应用 | 12 | 10 | 150 | $1000 |
| 高级系统 | 系统 | 28 | 22 | 700 | $1800 |
| 高级产品 | 应用 | 25 | 20 | 600 | $1700 |

### 岗位实例化

生成具体JobListing时，各门槛在基础值上 ±15% 随机浮动（取整），周薪同样 ±15%。

```
示例：初级后端 [赛博科技]
  简历门槛：14 → 随机12~16
  面试门槛：10 → 随机9~12
  竞争力门槛：180 → 随机153~207
  周薪：$1100 → 随机$935~$1265
```

## 五、求职判定流程

### 第一步：投简历 → 简历筛选

```
如果 你的简历分 ≥ 岗位简历门槛:
    通过筛选，进入面试
如果 你的简历分 < 岗位简历门槛:
    不通过
```

**确定性判定**，没有随机。玩家投之前就能100%知道自己能不能过简历关。

### 第二步：面试 → 面试关

```
如果 你的面试分 < 岗位面试门槛:
    直接失败（"基础不达标"）
如果 你的面试分 ≥ 岗位面试门槛:
    进入竞争环节
```

面试分达标也是**确定性**的。

### 第三步：竞争 → 拿offer

```
你的竞争力 = 简历分 × 面试分
对手竞争力 = 岗位竞争力门槛 × (0.7 ~ 1.3 随机)

如果 你的竞争力 ≥ 对手竞争力:
    拿到offer!
否则:
    "您表现优秀，但有更匹配的候选人"
```

**随机性只在最后一步**，而且来自"对手强弱"而非"你自己发挥"。

### 特殊修正

| 条件 | 效果 |
|------|------|
| 市场寒冬 | 对手竞争力 ×1.2（竞争更激烈） |
| 热门技能匹配 | 对手竞争力 ×0.85（供不应求，竞争缓和） |
| 批量投递2份 | 你的竞争力 ×0.85 |
| 批量投递3份 | 你的竞争力 ×0.70 |
| 内推 | 简历门槛视为 ×0.7（降低30%） |

|------|------|------|

### 6.3 行动按钮 —— 显示分数增量

每个行动按钮上直接标注**这次行动值多少分**：

```
── 成长 ──
[看文档·系统开发]  -1EP  简历+1~2    （系统 Lv.4, 进度 1.5/3）
[看文档·应用开发]  -1EP  简历+1~2    （应用 Lv.2, 进度 0/1）
[看文档·面试技巧]  -1EP  面试+1~3    （面试 Lv.3, 进度 2/2 → 升级!）
[看文档·英语]      -1EP  简历+1 面试+1 （英语 Lv.2, 进度 0/1）
[看文档·C++]       -1EP  简历+1      （C++ Lv.2, 进度 1/1 → 升级!）
[做开源·系统]      -2EP  简历+3 面试+2 （作品集进度 1/2）
[模拟面试]         -2EP  面试+3(本周)
```

说明：
- `简历+1~2`：学习获得XP，如果刚好升级则+2（因为等级×2），否则本次不直接加分但进度在涨
- 关键信息：如果即将升级，显示 `→ 升级!` 提示，这是最关键的决策信息

### 6.4 投简历界面

替代当前的概率显示：

```
── 投简历（本周可投3份，批量惩罚：2份-15%竞争力，3份-30%）──

☐ 外包测试 [码农互联] $810
  简历 ✓(17≥6)  面试 ✓(14≥4)  竞争力 238 vs ~22 → 稳拿

☐ 初级后端 [赛博科技] $1050
  简历 ✓(17≥15)  面试 ✓(14≥11)  竞争力 238 vs ~195 → 有戏

☐ 高级系统 [极客云] $1920
  简历 ✗(17<30)  ← 简历不过，无法投递
```

简历不达标的岗位**直接灰掉不可选**（而非让玩家投了再被拒）。这减少了无意义的操作，也让"简历分不够"的反馈更直接。

### 6.5 结算通知

面试结果改为显示具体分数对比：

```
✅ 初级后端 [赛博科技]：你的竞争力 238，击败对手(201)，获得offer！周薪 $1050
❌ 全栈开发 [量子前端]：你的竞争力 238，不敌对手(267)。差 29 分。
❌ 高级系统 [极客云]：面试分不达标(14<24)，未通过面试。
```

失败时显示**差多少分**，让玩家知道要补多少。

## 八、为什么乘法有效 —— Build 分化示例

### Build A: "面试流"

重点练面试技巧到Lv.7（面试分=21），搭配得体衬衫(+3)、面霸(+4)→面试分=28+。
简历分只堆到15左右。竞争力=15×28=420，够拿中级岗(180)。

**策略**：简历刚好够线就行，面试分溢出来碾压。

### Build B: "简历流"

系统技能Lv.8(16分)+C++ Lv.5(5分)+作品集2(6分)+英语3(3分)=简历30+。
面试只练到Lv.3(9分)。竞争力=30×9=270，也够中级岗。

**策略**：简历强到面试门槛刚过就行，靠竞争力硬碰。

### Build C: "社交流"

人脉拉满(简历+4, 面试+5)，作品集双方向(简历+9, 面试+6)，工具全社交线。
技能只需中等水平。竞争力靠全面加成堆出来。

**策略**：分散但每项都有加成，跟真实世界"人脉广路子野"的程序员一样。

### 关键点

三种Build的竞争力数值可能差不多，但**达到方式完全不同**，对应不同的行动选择和资源分配。而且因为乘法关系，极端偏科一定输给合理分配——但"合理"的定义取决于你的Build。

## 九、与现有系统的兼容

### 需要改的

1. `calc_resume_pass_rate()` → 改为 `calc_resume_score()` 返回整数分
2. `calc_interview_pass_rate()` → 改为 `calc_interview_score()` 返回整数分
3. 新增 `calc_competitiveness()` = 简历分 × 面试分
4. `action_apply_batch()` 的判定逻辑：概率判定 → 分数对比
5. `action_interview()` 的判定逻辑：概率判定 → 分数对比 + 对手随机
6. `JobListing` 新增字段：`resume_threshold`, `interview_threshold`, `compete_threshold`
7. `main_ui.gd` 全面改造显示

### 不需要改的

- 技能XP升级系统（底层不变，只是表现层翻译成分数）
- 工具/特质获取条件和触发机制
- 随机事件系统
- 经济系统（现金/生活费/零工）
- 岗位生成和市场刷新
