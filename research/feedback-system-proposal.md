# 数值反馈系统改造提案

> 状态：待讨论
> 日期：2026-03-24

## 问题诊断

当前游戏的核心体验问题：**因果链太长，反馈太抽象**。

### 当前的信息流

```
学习(+1XP) → 升级(Lv.3→4) → 通过率(55%→59%) → 随机判定 → 通过/失败
```

玩家做一个动作，要经过4步间接转换才看到结果，且最终结果是概率的。"升了一级"到底有多大用处？答案是"通过率从55%变成59%"——但4%是什么感觉？没感觉。

### V3引入工具/特质后加剧的问题

- 维度多了（技能×5 + 人脉 + 作品集 + 工具×12 + 特质×12），但每个维度的效果依然不直观
- 缺乏循序渐进感——无法感受到"我在变强"
- 导致玩家"一脸懵，随便选"

### 对比其他游戏

- **Slay the Spire**：打出攻击牌 → 敌人血条立掉。因果链=1步，确定性。
- **Into the Breach**：完美信息，你看到的就是会发生的。
- **当前游戏**：学一次技能 → ???（4步之后可能有用）

## 提案：两层改造

### 第一层：把概率翻译成人话

所有面向玩家的概率显示，从数字变成**定性标签**：

| 概率区间 | 标签 | 颜色 |
|---------|------|------|
| ≥80% | 十拿九稳 | 绿色 |
| 60-79% | 较有把握 | 浅绿 |
| 40-59% | 五五开 | 黄色 |
| 20-39% | 希望渺茫 | 橙色 |
| <20% | 基本无望 | 红色 |

投简历界面从：
```
简历≈62% 面试≈38%
```
变成：
```
简历：较有把握 | 面试：希望渺茫
```

**效果**：玩家看一眼就知道——"简历能过，但面试还差得远，应该去练面试技巧"。

### 第二层：岗位匹配度作为核心仪表盘

当前状态面板显示的是**原材料**（5个技能等级），玩家需要自己做心算把原材料转换成"我能不能拿到这个工作"。

改成直接显示**结果**：在主界面增加一个"岗位匹配"区域。

```
── 岗位匹配 ──
初级岗（外包测试/兼职前端）：  简历 十拿九稳 | 面试 五五开
中级岗（初级后端/全栈开发）：  简历 希望渺茫 | 面试 基本无望
高级岗（高级系统/产品工程师）：简历 基本无望 | 面试 基本无望
```

**效果**：每一个行动都能直观映射到仪表盘的变化。学了一次系统开发，回来看——"中级岗简历从'基本无望'变成了'希望渺茫'了，有进步"。

### 两层改造加起来的效果

**改造前的决策体验：**
> "我系统Lv.3，面试Lv.2，投这个岗简历62%面试38%……所以总概率是……23%？还是先练技能？练哪个？练了有多大用？……算了随便选一个吧。"

**改造后的决策体验：**
> "中级岗简历'希望渺茫'，面试'基本无望'。简历差在哪？技能差2级。那我先把系统练上去。面试差在哪？面试技巧太低。但面试得简历先过才有用，所以先搞简历。"

**不需要任何心算，决策自然浮现。**

## 实现范围

- 主要改 `main_ui.gd` 的显示层
- 底层数值系统（game_state.gd, game_data.gd）不需要改动
- 需要新增：岗位匹配度计算函数（聚合现有数值，按岗位层级输出定性标签）

## 待讨论

- 定性标签的具体措辞是否合适？
- 岗位匹配度是按三级（初/中/高）还是按具体岗位显示？
- 是否需要在行动按钮旁标注"做了这个之后匹配度会怎么变"的预览？
- 工具/特质的效果是否也需要类似的定性化处理？
