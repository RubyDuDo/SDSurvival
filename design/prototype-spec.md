# 原型版本需求设计

## 1. 目标

验证核心循环：**能量分配 + 技能方向选择 + 现金压力** 是否能产生有意义的两难抉择。

本版本以纯文字 UI 为主，不追求视觉效果。

---

## 2. 游戏流程

```
游戏开始 → [每周回合] → 12 周结束 / 现金归零
                |
                v
        显示当前状态（现金、技能、在职情况）
                |
                v
        分配能量点（选择本周做什么）
                |
                v
        结算（扣生活费、发工资、处理投递/面试结果）
                |
                v
            下一周
```

---

## 3. 核心资源

| 资源 | 说明 |
|------|------|
| **能量** | 每周 7 点，用于执行行动。在职时部分能量被锁定 |
| **现金** | 初始 $1200。每周扣除固定生活费。归零 = Game Over |
| **技能** | 两个方向各自独立的等级（0-10） |

---

## 4. 技能方向

只设 2 个方向，代表两条不同的职业路线：

| 技能 | 代表含义 |
|------|---------|
| **系统开发** | C++/底层/后端，对应稳定高薪但门槛高的岗位 |
| **应用开发** | 前端/全栈/业务开发，对应门槛较低但竞争激烈的岗位 |

另设 1 个**通用技能**：

| 技能 | 代表含义 |
|------|---------|
| **面试技巧** | 影响面试通过率，对所有岗位生效 |

> 为什么这样设计：2 个方向足以产生"专精 vs 分散"的抉择；通用技能提供第三个投资维度。

---

## 5. 行动列表

玩家每周将 7 点能量逐点分配到以下行动：

| 行动 | 能量消耗 | 效果 |
|------|---------|------|
| **学习系统开发** | 当前等级+1 | 系统开发技能 +1（升到第 N 级需要 N 点能量） |
| **学习应用开发** | 当前等级+1 | 应用开发技能 +1（升到第 N 级需要 N 点能量） |
| **练习面试** | 当前等级+1 | 面试技巧 +1（升到第 N 级需要 N 点能量） |
| **兼职零工** | 3 | 获得 $280 |
| **全职零工** | 5 | 获得 $520 |
| **投递简历** | 1 | 选择一个岗位投递，根据技能匹配度判定是否获得面试机会 |
| **参加面试** | 2 | 对已获得面试机会的岗位进行面试，根据综合能力判定是否拿到 offer |

---

## 6. 岗位设计

### 6.1 岗位列表

**系统开发方向：**

| 岗位 | 技能要求 | 周薪 | 占用能量 |
|------|---------|------|---------|
| 外包测试 | 系统 3 | $780/周 | 4 |
| 初级后端 | 系统 5 | $1100/周 | 5 |
| 高级系统工程师 | 系统 8 | $1800/周 | 5 |

**应用开发方向：**

| 岗位 | 技能要求 | 周薪 | 占用能量 |
|------|---------|------|---------|
| 兼职前端 | 应用 3 | $680/周 | 3 |
| 全栈开发 | 应用 5 | $1000/周 | 5 |
| 高级产品工程师 | 应用 8 | $1700/周 | 5 |

### 6.2 设计意图

- **零工**：兼职零工（3能量/$280）不够覆盖生活费；全职零工（5能量/$520）勉强保活但几乎无余力成长
- **低 tier 岗位**：薪资约为全职零工的 1.5 倍，门槛适中，提供现金续命同时有少量成长空间
- **高 tier 岗位**：高收入（约零工 3.5 倍），但需要长期投入技能才能够到
- **兼职前端**只占 3 能量——比外包测试（占 4）轻松，但收入也更低。提供不同的生存策略选择
- 两个方向的高级岗位收入接近，没有"绝对正确"的方向
- 原型阶段所有 6 个岗位常驻可见（后续版本加入刷新机制）

### 6.3 在职规则

- 同一时间只能持有 1 份工作
- 在职时每周自动发工资，但对应能量被锁定（不可自由分配）
- **在职期间可以求职**（用剩余自由能量投简历、参加面试）
- 可以选择辞职（**下周生效**：辞职当周仍在职、仍拿工资、能量仍被占用；下周恢复全部能量）

---

## 7. 求职流程

求职是一个**跨越多周的异步流程**，模拟真实求职中的等待和不确定性。

### 7.1 流程时间线

```
第 N 周：投递简历（消耗 1 能量）
            ↓
第 N+1 周结算时：收到通知 —— 获得面试机会 or 被拒
            ↓
第 N+1 周（或之后）：参加面试（消耗 2 能量）
            ↓
第 N+2 周结算时：收到面试结果 —— 获得 offer or 被拒
            ↓
获得 offer 后：选择接受/拒绝（offer 有效期 2 周）
```

### 7.2 投递

- 消耗 1 能量，选择一个岗位投递
- 每个岗位同一时间只能有一个进行中的申请（投递中 / 待面试 / 已面试待结果）
- 被拒后可以立即重新投递同一岗位
- **结果在下周结算时通知**：技能 >= 岗位要求 → 获得面试机会；否则拒信

### 7.3 面试

- 消耗 2 能量，对持有面试机会的岗位进行面试
- **结果在下周结算时通知**
- 通过率 = 基础 35% + (技能 - 岗位要求) × 8% + 面试技巧 × 2%
- 通过率上限 95%，下限 20%
- 通过 → 获得 offer；失败 → 需重新投递

### 7.4 Offer

- 可以同时持有多个 offer
- **offer 有效期 2 周**，过期自动作废
- 接受 offer → 下周开始上班（若当前在职，视为跳槽，当前工作自动结束）
- 拒绝 offer → 该岗位可重新投递

---

## 8. 结算规则

每周结束时按以下顺序结算：

1. 若在职 → 发放周薪
2. 扣除生活费（固定 $500/周）
3. 处理求职通知：
   - 上周投递的简历 → 本周出结果（获得面试机会 or 拒信）
   - 上周参加的面试 → 本周出结果（获得 offer or 拒信）
4. Offer 有效期 -1 周，过期的 offer 自动作废
5. 若现金 < 0 → Game Over
6. 推进到下一周

---

## 9. 结局

12 周结束时，根据最终状态判定结局：

| 条件 | 结局 | 描述 |
|------|------|------|
| 持有高级岗位 offer 或在职 | **上岸成功** | "你终于拿到了理想的工作。" |
| 持有中级岗位在职 | **站稳脚跟** | "不算完美，但你活下来了。" |
| 持有低 tier 岗位或失业但现金 > 0 | **艰难求生** | "还在挣扎，但没有放弃。" |
| 中途现金归零 | **Game Over** | "你的积蓄耗尽了……" |

## 12. 随机事件系统

### 12.1 设计意图

每周结算时有概率触发一个随机事件，制造不可预测的压力和惊喜，提升重玩差异性。玩家需要在策略执行中保留容错空间，而不能只按最优路线走。

### 12.2 触发规则

- 每周结算时触发概率：**30%**
- 同一周最多触发 **1 个**事件
- 部分事件有触发前提（如"朋友内推"需要人脉点数 > 0）
- 事件分为**负面 / 正面 / 中性**三类，权重比约 **4 : 3 : 3**

### 12.3 事件列表

**负面事件（权重 40%）**

| 事件名 | 触发条件 | 效果 |
|--------|---------|------|
| 电脑出了点问题 | 无 | 支出 $300 维修费 |
| 临时涨租 | 无 | 本周额外支出 $250 |
| 突发低烧 | 无 | 本周最大能量 -2 |
| 面试被放鸽子 | 有进行中的面试 | 随机取消一个"待面试"状态的申请（此事件在事件池中权重极低，每次触发概率约 3%，确保单个申请被取消的概率 ≤ 5%） |
| 简历石沉大海（加速版） | 有进行中的投递 | 随机取消一个"已投递"状态的申请 |
| 被同学安利了一门课 | 无 | 本周学习任意技能消耗能量 +1（每次） |

**正面事件（权重 30%）**

| 事件名 | 触发条件 | 效果 |
|--------|---------|------|
| 接到外包小单 | 无 | 额外获得 $350 |
| 今天状态奇好 | 无 | 本周能量上限 +2 |
| 找到了省钱攻略 | 无 | 本周生活费减半 |
| 灵感爆发 | 无 | 本周学习技能获得双倍 XP |
| 朋友内推 | 人脉 ≥ 1 | 随机选一个当期市场岗位，简历通过率 ×1.5（不是直接通过，仍需投递流程）；人脉点数越高，此事件权重越大，且可以指定方向；**不影响面试阶段判定** |

**中性事件（权重 30%）**

| 事件名 | 触发条件 | 效果 |
|--------|---------|------|
| 某公司暂停招聘 | 无 | 当期市场随机一个岗位本周不可新投递 |
| 行业新闻刷屏 | 无 | 展示"市场资讯"文本，下次岗位刷新中该方向岗位 +1 |
| 面试官今天心情很好 | 本周有已参加面试待结果 | 本次面试好感判定通过率 +20% |
| 熬夜刷题 | 无 | 下周能量上限 -1，但本周学习任意技能 XP +2 |

### 12.4 UI 呈现

- 事件在**周结算界面**中显示，位于收入/支出之后、求职通知之前
- 格式：`🎲 本周事件：[事件名] — [效果描述]`
- 没有事件时不显示该行

## 13. 人脉系统

### 13.1 设计意图

人脉是第三条独立的资源投入路线（与技能、作品集并列），专注于降低"运气成分"中的求职门槛。人脉值越高，内推事件触发越频繁，且内推质量更高（可指定方向）。但人脉是机会型收益，不是稳定收益，不能替代技能。

### 13.2 获取方式

| 行动 | 能量消耗 | 效果 |
|------|---------|------|
| **维护人脉** | 1 | 人脉点数 +1（上限 10）|

- 人脉没有 XP 累积机制，每次 1 能量 = +1 点，直接叠加
- 每周可以多次维护人脉（视能量情况）

### 13.3 人脉的效果

| 人脉点数 | 效果 |
|---------|------|
| 1–3 | "朋友内推"事件权重 ×1.5，内推岗位随机（不可选方向） |
| 4–6 | "朋友内推"事件权重 ×2.5，内推时可选方向（系统 / 应用） |
| 7–9 | "朋友内推"事件权重 ×4，内推时可选具体岗位层级（低/中/高） |
| 10 | 额外解锁"打听内部情报"行动（见下） |

### 13.4 打听内部情报（人脉满级解锁）

- **行动**：打听内部情报（消耗 1 能量）
- **效果**：提前得知下次岗位市场刷新的方向偏向（系统偏 / 应用偏 / 均衡）
- 每局限用 2 次

### 13.5 "朋友内推"事件的基础权重

随机事件池中，"朋友内推"的权重计算如下：

```
base_weight = 0（人脉为0时不进入事件池）
若人脉 >= 1：base_weight = 3 + 人脉点数 × 乘数（见13.3）
```

## 14. 市场寒冬

### 14.1 设计意图

在游戏中后段制造一次全局性压力事件，迫使玩家应对突发的市场收缩，检验资金储备和多向布局是否扎实。

### 14.2 触发规则

- **触发窗口**：第 5–9 周之间
- **触发概率**：进入触发窗口后，每次岗位市场刷新（第 7 周）时有 **30%** 概率触发
- **持续时长**：1 个刷新周期（3 周），第 10 周恢复正常
- 每局最多触发 **1 次**

### 14.3 寒冬期效果

| 项目 | 正常期 | 寒冬期 |
|------|-------|-------|
| 每次市场刷新的岗位数量 | 5 个 | 3 个 |
| 面试第三层竞争系数 | 正常 | 各层级 +15% |
| 零工薪资 | 正常 | 下调 10% |
| 可触发的负面随机事件权重 | 正常 | ×1.3 |

### 14.4 UI 提示

- 寒冬期间界面顶部显示：`📉 市场下行中（剩余 X 周）`
- 岗位市场刷新时通知：`⚠️ 行业寒冬来袭！本期市场岗位减少，竞争更为激烈。`
- 寒冬结束时通知：`📈 市场回暖，岗位供给恢复正常。`

## 15. 作品集系统

### 15.1 设计意图

作品集是一种"花时间换简历加成"的投资路线，适合技能偏低但想冲更高岗位的玩家。与技能成长不同，作品集的收益是**直接提升简历通过率**而非能力本身，代表"包装"而非"实力"。这也制造了一种张力：作品集可以骗过简历关，但面试关仍然考真实能力。

### 15.2 获取方式

| 行动 | 能量消耗 | 效果 |
|------|---------|------|
| **做个人项目（系统方向）** | 3 | 系统方向项目点数 +1 |
| **做个人项目（应用方向）** | 3 | 应用方向项目点数 +1 |

- 两个方向独立积累，互不影响
- 每个方向有效项目点数上限：**3 个**（超过 3 个不再增加效果）
- 不需要 XP 累积，每次 3 能量直接 +1

### 15.3 作品集的效果

**简历阶段（加成叠加在简历通过率上）：**

| 方向项目点数 | 简历通过率加成 |
|------------|-------------|
| 0 | +0% |
| 1 | +8% |
| 2 | +16% |
| 3 | +24% |

**面试阶段（影响第二层好感判定）：**

| 方向项目点数 | 面试好感通过率倍率 |
|------------|----------------|
| 0 | ×1.00（无影响） |
| 1 | ×1.08 |
| 2 | ×1.15 |
| 3 | ×1.20 |

- 作品集体现真实个人产出，面试官看到具体项目会产生好感，因此同时影响简历和面试
- 两项加成仅对**对应方向**的岗位生效（系统项目不影响应用岗位）
- 作品集**不影响第三层竞争判定**——项目再好也不能决定别人的水平

**被追问风险（"包装被戳穿"）：**

- 触发条件：主技能比岗位要求低 2 级以上，且作品集点数 ≥ 2
- 触发概率：10%
- 效果：面试好感倍率改为 ×0.90（追问时答不上来，印象反而变差）
- 这就是作品集面试加成范围 90%–120% 的由来：实力够时稳定加分，靠包装硬冲偶尔翻车

### 15.4 设计上的权衡

- 3 能量/次，比学习技能（1 能量）贵；但简历和面试两端都有收益
- 适合已经有一定技能基础、想提升求职成功率的中后期投入
- 与技能的区别：技能是硬实力（影响更大、更全面），作品集是展示加成（见效快但有上限）；两者互补
